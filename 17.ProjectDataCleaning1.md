## 🔹 Create Staging Table (Working Copy)

```sql id="dc3"
-- Create a copy of the original table
CREATE TABLE layoffs_staging
LIKE layoffs;
```

```sql id="dc4"
-- Insert data into staging table
INSERT INTO layoffs_staging
SELECT *
FROM layoffs;
```

👉 Best practice: Always work on a copy, not the original data

## 🔹 Identify Duplicates

```sql id="dc5"
SELECT *,
ROW_NUMBER() OVER (
    PARTITION BY company,
                 industry,
                 total_laid_off,
                 percentage_laid_off,
                 `date`
) AS row_num
FROM layoffs_staging;
```

👉 `ROW_NUMBER()` assigns a number to duplicate rows

---

## 🔹 Find Duplicate Rows using CTE

```sql id="dc6"
WITH duplicate_cte AS (
    SELECT *,
    ROW_NUMBER() OVER (
        PARTITION BY company,
                     location,
                     industry,
                     total_laid_off,
                     percentage_laid_off,
                     `date`,
                     stage,
                     country,
                     funds_raised_millions
    ) AS row_num
    FROM layoffs_staging
)
SELECT *
FROM duplicate_cte
WHERE row_num > 1;
```

👉 Rows with `row_num > 1` are duplicates

## 🔹 Remove Duplicates (Using New Table)

```sql id="dc8"
-- Create new table with additional row_num column
CREATE TABLE layoffs_staging2 (
    company TEXT,
    location TEXT,
    industry TEXT,
    total_laid_off INT DEFAULT NULL,
    percentage_laid_off TEXT,
    date TEXT,
    stage TEXT,
    country TEXT,
    funds_raised_millions INT DEFAULT NULL,
    row_num INT
);
```

---

```sql id="dc9"

INSERT INTO layoffs_staging2
SELECT *,
ROW_NUMBER() OVER (

                 total_laid_off,
                 percentage_laid_off,
                 `date`,
                 stage,
                 country,
                 funds_raised_millions
```

## 🔹 Standardizing Data

### 📌 Clean Company Names

```sql id="dc13"
-- Remove extra spaces
SELECT company, TRIM(company)
FROM layoffs_staging2;
```

```sql id="dc14"
UPDATE layoffs_staging2
SET company = TRIM(company);
```

---

### 📌 Standardize Industry

```sql id="dc15"
SELECT DISTINCT industry
FROM layoffs_staging2;
```

```sql id="dc16"
-- Convert similar values to one category
UPDATE layoffs_staging2
SET industry = 'Crypto'
WHERE industry LIKE 'Crypto%';
```

---

```sql id="dc17"
-- Count records per industry
SELECT industry, COUNT(industry)
FROM layoffs_staging2
GROUP BY industry
ORDER BY industry;
```

## 🔹 Fix Date Format

```sql id="dc24"
-- Convert text to DATE format
UPDATE layoffs_staging2
SET date = STR_TO_DATE(date, '%m/%d/%Y');
```

```sql id="dc25"
-- Change column datatype to DATE
ALTER TABLE layoffs_staging2
MODIFY COLUMN date DATE;
```

## 🧠 Key Notes

* Always create a **staging table** before cleaning
* Use `ROW_NUMBER()` to identify duplicates
* Standardize text data using `TRIM`, `UPDATE`, `LIKE`
* Fix inconsistent values manually where needed
* Convert date formats properly
* Verify changes after each step
