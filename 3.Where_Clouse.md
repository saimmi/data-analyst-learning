## 🔹 WHERE Clause (All Syntax + Examples)

---

## 📌 Basic Syntax

```sql id="syn1"
SELECT column_name(s)
FROM table_name
WHERE condition;
```

👉 `WHERE` is used to filter rows based on a condition

---

## 🔹 1️⃣ Select All Columns with WHERE

```sql id="syn2"
SELECT *
FROM employee_salary
WHERE first_name = 'Leslie';
```

👉 `*` → selects all columns
👉 `=` → equality operator

---

## 🔹 2️⃣ Comparison Operators

### ✅ Greater Than

```sql id="syn3"
SELECT *
FROM employee_salary
WHERE salary > 50000;
```

---

### ✅ Greater Than or Equal

```sql id="syn4"
SELECT *
FROM employee_salary
WHERE salary >= 50000;
```

---

### ✅ Less Than

```sql id="syn5"
SELECT *
FROM employee_salary
WHERE salary < 50000;
```

---

### ✅ Less Than or Equal

```sql id="syn6"
SELECT *
FROM employee_salary
WHERE salary <= 50000;
```

---

### ✅ Not Equal

```sql id="syn7"
SELECT *
FROM employee_demographics
WHERE gender != 'Female';
```

👉 Alternative:

```sql
WHERE gender <> 'Female';
```

---

## 🔹 3️⃣ Filtering by Text

```sql id="syn8"
SELECT *
FROM employee_demographics
WHERE gender = 'Male';
```

👉 String values must be inside `' '`

---

## 🔹 4️⃣ Filtering by Date

```sql id="syn9"
SELECT *
FROM employee_demographics
WHERE birth_date > '1985-01-01';
```

👉 Format: `'YYYY-MM-DD'`

---

## 🔹 5️⃣ Logical Operators

---

### ✅ AND (Both conditions must be TRUE)

```sql id="syn10"
SELECT *
FROM employee_demographics
WHERE gender = 'Male'
  AND age > 40;
```

---

### ✅ OR (Any one condition TRUE)

```sql id="syn11"
SELECT *
FROM employee_demographics
WHERE gender = 'Male'
   OR age > 40;
```

---

### ✅ NOT (Negates condition)

```sql id="syn12"
SELECT *
FROM employee_demographics
WHERE NOT gender = 'Male';
```

---

### ✅ Combined Conditions

```sql id="syn13"
SELECT *
FROM employee_demographics
WHERE (first_name = 'Leslie' AND age = 44)
   OR age > 55;
```

👉 `()` controls execution priority

---

## 🔹 6️⃣ LIKE Operator (Pattern Matching)

### 📌 Basic Syntax

```sql id="syn14"
SELECT column_name
FROM table_name
WHERE column_name LIKE pattern;
```

---

### ✅ Starts With

```sql id="syn15"
SELECT *
FROM employee_demographics
WHERE first_name LIKE 'Jer%';
```

👉 `%` → any number of characters

---

### ✅ Ends With

```sql id="syn16"
SELECT *
FROM employee_demographics
WHERE first_name LIKE '%a';
```

---

### ✅ Contains

```sql id="syn17"
SELECT *
FROM employee_demographics
WHERE first_name LIKE '%er%';
```

---

## 🔹 7️⃣ Wildcards

---

### ✅ `%` → Any number of characters

```sql id="syn18"
SELECT *
FROM employee_demographics
WHERE first_name LIKE 'a%';
```

---

### ✅ `_` → Exactly one character

```sql id="syn19"
SELECT *
FROM employee_demographics
WHERE first_name LIKE 'a__';
```

---

### ✅ Combination

```sql id="syn20"
SELECT *
FROM employee_demographics
WHERE first_name LIKE 'a__%';
```

---

## 🔹 8️⃣ LIKE with Dates

```sql id="syn21"
SELECT *
FROM employee_demographics
WHERE birth_date LIKE '1989%';
```

👉 Filters all records from year 1989

---

## 🔹 9️⃣ Multiple Conditions (Real Example)

```sql id="syn22"
SELECT *
FROM employee_demographics
WHERE birth_date > '1985-01-01'
   OR gender = 'Male';
```

---

```sql id="syn23"
SELECT *
FROM employee_demographics
WHERE birth_date > '1985-01-01'
   OR NOT gender = 'Male';
```

---

## 🔹 🔟 Case Sensitivity Note

```sql id="syn24"
SELECT *
FROM employee_demographics
WHERE gender = 'Male';
```

👉 Use exact values as stored in table
👉 `'male'` may not match `'Male'` depending on settings

---

## 🧠 Quick Revision Summary

* `WHERE` → filters rows
* `=` → equal
* `!=` / `<>` → not equal
* `>` `<` `>=` `<=` → comparisons
* `AND` → both true
* `OR` → any true
* `NOT` → reverse condition
* `LIKE` → pattern matching
* `%` → any characters
* `_` → single character
