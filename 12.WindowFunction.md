## 🔹 Window Functions (All Syntax + Examples)

---

## 📌 What is a Window Function?

* Performs calculations **across a set of rows**
* Does **NOT collapse rows** (unlike `GROUP BY`)
* Each row still remains visible

---

## 📌 Basic Syntax

```sql id="wfn1"
SELECT column_name,
       window_function() OVER (PARTITION BY column_name ORDER BY column_name)
FROM table_name;
```

👉 `OVER()` defines the window
👉 `PARTITION BY` → groups data (like GROUP BY but keeps rows)
👉 `ORDER BY` → defines order inside each partition

---

## 🔹 1️⃣ JOIN (Base for Window Functions)

```sql id="wfn2"
SELECT *
FROM employee_demographics dem
JOIN employee_salary sal
  ON dem.employee_id = sal.employee_id;
```

👉 Combines both tables for analysis

---

## 🔹 2️⃣ GROUP BY vs WINDOW FUNCTION

---

### ✅ Using GROUP BY

```sql id="wfn3"
SELECT gender, AVG(salary) AS avg_sal
FROM employee_demographics dem
JOIN employee_salary sal
  ON dem.employee_id = sal.employee_id
GROUP BY gender;
```

👉 Returns **one row per gender**

---

### ✅ Using WINDOW FUNCTION

```sql id="wfn4"
SELECT gender, AVG(salary) OVER (PARTITION BY gender) AS avg_sal
FROM employee_demographics dem 
JOIN employee_salary sal
  ON dem.employee_id = sal.employee_id;
```

👉 Returns **all rows + average repeated per gender**
👉 Does NOT group rows

---

## 🔹 3️⃣ GROUP BY with More Columns

```sql id="wfn5"
SELECT dem.first_name, dem.last_name, gender, AVG(salary) AS avg_sal
FROM employee_demographics dem
JOIN employee_salary sal
  ON dem.employee_id = sal.employee_id
GROUP BY dem.first_name, dem.last_name, gender;
```

👉 Aggregation per employee

---

## 🔹 4️⃣ Running Total (SUM OVER)

```sql id="wfn6"
SELECT dem.employee_id, dem.first_name, dem.last_name, gender, salary,
       SUM(salary) OVER (PARTITION BY gender ORDER BY dem.employee_id) AS rolling_total
FROM employee_demographics dem
JOIN employee_salary sal 
  ON dem.employee_id = sal.employee_id;
```

👉 Calculates **cumulative (running) total**
👉 Reset happens for each gender

---

## 🔹 5️⃣ ROW_NUMBER()

---

### ✅ Basic

```sql id="wfn7"
SELECT dem.employee_id, dem.first_name, dem.last_name, gender, salary,
       ROW_NUMBER() OVER()
FROM employee_demographics dem
JOIN employee_salary sal
  ON dem.employee_id = sal.employee_id;
```

👉 Assigns unique row numbers (no partition)

---

### ✅ With PARTITION

```sql id="wfn8"
SELECT dem.employee_id, dem.first_name, dem.last_name, gender, salary,
       ROW_NUMBER() OVER (PARTITION BY gender)
FROM employee_demographics dem
JOIN employee_salary sal
  ON dem.employee_id = sal.employee_id;
```

👉 Row numbers restart for each gender

---

### ✅ With ORDER BY

```sql id="wfn9"
SELECT dem.employee_id, dem.first_name, dem.last_name, gender, salary,
       ROW_NUMBER() OVER (PARTITION BY gender ORDER BY salary DESC)
FROM employee_demographics dem
JOIN employee_salary sal
  ON dem.employee_id = sal.employee_id;
```

👉 Highest salary gets row number = 1

---

## 🔹 6️⃣ RANK() vs DENSE_RANK()

---

```sql id="wfn10"
SELECT dem.employee_id, dem.first_name, dem.last_name, gender, salary,
       ROW_NUMBER() OVER (PARTITION BY gender ORDER BY salary DESC) AS row_num,
       RANK() OVER (PARTITION BY gender ORDER BY salary DESC) AS rank_num
FROM employee_demographics dem
JOIN employee_salary sal
  ON dem.employee_id = sal.employee_id;
```

---

```sql id="wfn11"
SELECT dem.employee_id, dem.first_name, dem.last_name, gender, salary,
       ROW_NUMBER() OVER (PARTITION BY gender ORDER BY salary DESC) AS row_num,
       RANK() OVER (PARTITION BY gender ORDER BY salary DESC) AS rank_num,
       DENSE_RANK() OVER (PARTITION BY gender ORDER BY salary DESC) AS dense_rank_num
FROM employee_demographics dem
JOIN employee_salary sal
  ON dem.employee_id = sal.employee_id;
```

---

### 🔍 Difference

| Function     | Handles Ties | Skips Rank |
| ------------ | ------------ | ---------- |
| ROW_NUMBER() | ❌ No         | ❌ No       |
| RANK()       | ✅ Yes        | ✅ Yes      |
| DENSE_RANK() | ✅ Yes        | ❌ No       |

---

## 🧠 Key Concepts

* `OVER()` → defines window
* `PARTITION BY` → splits data into groups
* `ORDER BY` → defines ranking order
* Window functions **do not reduce rows**
* Useful for:

  * Ranking
  * Running totals
  * Analytics

---

## 🔁 GROUP BY vs WINDOW FUNCTION

| Feature        | GROUP BY | WINDOW FUNCTION |
| -------------- | -------- | --------------- |
| Row reduction  | ✅ Yes    | ❌ No            |
| Aggregation    | ✅ Yes    | ✅ Yes           |
| Keeps all rows | ❌ No     | ✅ Yes           |
