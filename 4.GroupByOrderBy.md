## 🔹 GROUP BY & ORDER BY (All Syntax + Examples)

---

## 📌 GROUP BY – Basic Syntax

```sql id="g1"
SELECT column_name, AGGREGATE_FUNCTION(column_name)
FROM table_name
GROUP BY column_name;
```

👉 `GROUP BY` is used to **group rows with same values**
👉 Mostly used with:

* `AVG()`
* `MAX()`
* `MIN()`
* `COUNT()`
* `SUM()`

---

## 🔹 1️⃣ Basic GROUP BY

```sql id="g2"
SELECT gender
FROM employee_demographics
GROUP BY gender;
```

👉 Returns **unique gender values**
👉 Same as `DISTINCT`, but used for grouping

---

## 🔹 2️⃣ GROUP BY with AVG()

```sql id="g3"
SELECT gender, AVG(age)
FROM employee_demographics
GROUP BY gender;
```

👉 Calculates **average age per gender**

---

## 🔹 3️⃣ GROUP BY on employee_salary

```sql id="g4"
SELECT occupation
FROM employee_salary
GROUP BY occupation;
```

👉 Shows unique occupations

⚠️ Note:

* "Office Manager" appears once because GROUP BY groups duplicate values

---

## 🔹 4️⃣ GROUP BY with salary

```sql id="g5"
SELECT salary
FROM employee_salary
GROUP BY salary;
```

👉 Groups rows based on salary values

---

## 🔹 5️⃣ GROUP BY Multiple Columns

```sql id="g6"
SELECT occupation, salary
FROM employee_salary
GROUP BY occupation, salary;
```

👉 Creates groups based on **combination of occupation + salary**
👉 Office Manager appears multiple times because salary differs

---

## 🔹 6️⃣ Aggregate Functions

---

### ✅ AVG + MAX

```sql id="g7"
SELECT gender, AVG(age), MAX(age)
FROM employee_demographics
GROUP BY gender;
```

---

### ✅ MIN

```sql id="g8"
SELECT gender, MIN(age)
FROM employee_demographics
GROUP BY gender;
```

---

### ✅ MAX

```sql id="g9"
SELECT gender, MAX(age)
FROM employee_demographics
GROUP BY gender;
```

---

### ✅ COUNT

```sql id="g10"
SELECT gender, COUNT(age)
FROM employee_demographics
GROUP BY gender;
```

---

## 🔹 7️⃣ Combined Aggregates

```sql id="g11"
SELECT gender, AVG(age), MAX(age), MIN(age)
FROM employee_demographics
GROUP BY gender;
```

---

```sql id="g12"
SELECT gender, AVG(age), MAX(age), MIN(age), COUNT(age)
FROM employee_demographics
GROUP BY gender;
```

👉 Best practice for analysis (all metrics together)

---

## 📌 Important Rule

❗ Every column in SELECT must be:

* either in `GROUP BY`
* or inside an aggregate function

---

## 🔹 ORDER BY – Basic Syntax

```sql id="o1"
SELECT column_name
FROM table_name
ORDER BY column_name [ASC | DESC];
```

👉 Used to **sort the result**

---

## 🔹 1️⃣ Default Sorting (ASC)

```sql id="o2"
SELECT *
FROM employee_demographics
ORDER BY first_name;
```

👉 Default = **ASC (ascending)**

---

```sql id="o3"
SELECT *
FROM employee_demographics
ORDER BY first_name ASC;
```

👉 Same as above (explicit ASC)

---

## 🔹 2️⃣ DESC Sorting

```sql id="o4"
SELECT *
FROM employee_demographics
ORDER BY first_name DESC;
```

👉 Sorts in **descending order**

---

## 🔹 3️⃣ Sort by Different Columns

```sql id="o5"
SELECT *
FROM employee_demographics
ORDER BY gender;
```

```sql id="o6"
SELECT *
FROM employee_demographics
ORDER BY age;
```

```sql id="o7"
SELECT *
FROM employee_demographics
ORDER BY age DESC;
```

---

## 🔹 4️⃣ Multiple Column Sorting

```sql id="o8"
SELECT *
FROM employee_demographics
ORDER BY gender, age;
```

👉 First sorts by `gender`, then by `age`

---

```sql id="o9"
SELECT *
FROM employee_demographics
ORDER BY gender, age DESC;
```

👉 Gender ascending + Age descending

---

```sql id="o10"
SELECT *
FROM employee_demographics
ORDER BY age, gender;
```

👉 First priority → `age`
👉 Second priority → `gender`

---

## 🔹 5️⃣ Column Position (Not Recommended)

```sql id="o11"
SELECT *
FROM employee_demographics
ORDER BY 5, 4;
```

👉 Sorts using column position:

* 5 → age
* 4 → gender

⚠️ Not recommended because:

* Reduces readability
* Can break if column order changes

---

## 🧠 Quick Revision Summary

### GROUP BY

* Groups rows with same values
* Used with aggregate functions
* Required for data analysis

### ORDER BY

* Sorts output
* Default → ASC
* Use DESC for reverse order
* Multiple columns = priority order
