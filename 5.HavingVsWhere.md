## 🔹 HAVING vs WHERE

---

## 📌 Difference Between WHERE and HAVING

* `WHERE` → filters rows **before grouping**
* `HAVING` → filters data **after GROUP BY (on aggregated values)**

---

## 📌 Basic GROUP BY Example

```sql id="hw1"
-- Average age by gender
SELECT gender, AVG(age)
FROM employee_demographics
GROUP BY gender;
```

---

## 📌 Using HAVING

```sql id="hw2"
-- Filter groups where average age > 40
SELECT gender, AVG(age)
FROM employee_demographics
GROUP BY gender
HAVING AVG(age) > 40;
```

👉 `HAVING` works with aggregate functions like `AVG()`, `SUM()`, `COUNT()`

---

## 📌 Salary Table

```sql id="hw3"
SELECT *
```

---

## 📌 GROUP BY with Salary

```sql id="hw4"
-- Average salary by occupation
SELECT occupation, AVG(salary)
FROM employee_salary
GROUP BY occupation;
```

---

## 📌 Using WHERE with GROUP BY

```sql id="hw5"
-- Filter rows before grouping
SELECT occupation, AVG(salary)
FROM employee_salary
WHERE occupation LIKE '%manager%'
GROUP BY occupation;
```

👉 `WHERE` filters only rows where occupation contains "manager"

---

## 📌 Using HAVING with GROUP BY

```sql id="hw6"
-- Filter groups after aggregation
SELECT occupation, AVG(salary)
FROM employee_salary
WHERE occupation LIKE '%manager%'
GROUP BY occupation
HAVING AVG(salary) > 75000;
```

👉 First filters rows → then groups → then applies HAVING condition

---

## 🧠 Key Notes

* `WHERE` cannot use aggregate functions
* `HAVING` is used with `GROUP BY`
* Execution order:

  1. `WHERE`
  2. `GROUP BY`
  3. `HAVING`

---

## 🎯 Simple Understanding

* `WHERE` → filter rows
* `HAVING` → filter grouped results
