## 🔹 Temporary Tables (TEMP TABLE)

---

## 📌 What is a Temporary Table?

* A temporary table stores data for a short time
* Exists only during the current session
* Automatically deleted after session ends

---

## 📌 Create Temporary Table

```sql id="nda4ft"
CREATE TEMPORARY TABLE temp_table (
    first_name VARCHAR(50),
    last_name VARCHAR(50),
    favorite_movie VARCHAR(100)
);
```

---

## 📌 View Data

```sql id="tj4ijn"
SELECT *
FROM temp_table;
```

---

## 📌 Insert Data

```sql id="m62b3d"
INSERT INTO temp_table
VALUES ('Alex', 'Freberg', 'Lord of the Rings: The Two Towers');
```

---

```sql id="ve7xqn"
SELECT *
```

## 📌 Using Existing Table

```sql id="ab4yeo"
SELECT *
```

## 📌 Create TEMP TABLE from Query

```sql id="5q3pt0"
CREATE TEMPORARY TABLE salary_over_50k AS
SELECT *
FROM employee_salary
WHERE salary >= 50000;
```

---

```sql id="s158sg"
SELECT *
FROM salary_over_50k;
```

## 📌 TEMP TABLE with JOIN

```sql id="ntdha3"
CREATE TEMPORARY TABLE temp_join AS
SELECT 
    dem.employee_id,
    dem.first_name,
    sal.salary
FROM employee_demographics dem
JOIN employee_salary sal
  ON dem.employee_id = sal.employee_id;
```

---

```sql id="4etrox"
SELECT *
FROM temp_join;
```

---

## 📌 Drop TEMP TABLE

```sql id="3kxt03"
DROP TEMPORARY TABLE IF EXISTS temp_employee;
```

---

## 🧠 Key Notes

* Exists only in current session
* Automatically deleted after session ends
* Useful for intermediate results
* Helps simplify complex queries
* Can be reused multiple times within session
