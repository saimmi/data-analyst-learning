## 🔹 JOINS

---

## 📌 View Tables

```sql id="j1"
SELECT *
FROM employee_demographics;

SELECT *
FROM employee_salary;
```

---

## 🔹 INNER JOIN

```sql id="j2"
-- Returns matching records from both tables
SELECT *
FROM employee_demographics
INNER JOIN employee_salary
  ON employee_demographics.employee_id = employee_salary.employee_id;
```

---

## 📌 Using Aliases (Best Practice)

```sql id="j3"
-- Short names improve readability
SELECT *
FROM employee_demographics AS dem
INNER JOIN employee_salary AS sal
  ON dem.employee_id = sal.employee_id;
```

---

## 📌 Selecting Specific Columns

```sql id="j4"
SELECT employee_demographics.employee_id, age, occupation
FROM employee_demographics
INNER JOIN employee_salary
  ON employee_demographics.employee_id = employee_salary.employee_id;
```

---

```sql id="j5"
-- Better readability with full column reference
SELECT 
    dem.employee_id,
    dem.age,
    sal.occupation
FROM employee_demographics dem
INNER JOIN employee_salary sal
  ON dem.employee_id = sal.employee_id;
```

---

## 🔹 LEFT JOIN

```sql id="j6"
-- Returns all rows from left table + matching from right
SELECT *
FROM employee_demographics
LEFT JOIN employee_salary
  ON employee_demographics.employee_id = employee_salary.employee_id;
```

---

## 🔹 RIGHT JOIN

```sql id="j7"
-- Returns all rows from right table + matching from left
SELECT *
FROM employee_demographics
RIGHT JOIN employee_salary
  ON employee_demographics.employee_id = employee_salary.employee_id;
```

---

## 🔹 SELF JOIN

```sql id="j8"
-- Join table with itself
SELECT *
FROM employee_demographics AS emp1
JOIN employee_demographics AS emp2
  ON emp1.employee_id + 1 = emp2.employee_id;
```

---

```sql id="j9"
-- Example: pairing employees (current with next ID)
SELECT 
    emp1.employee_id AS emp_id_1,
    emp1.first_name AS first_name_1,
    emp1.last_name AS last_name_1,
    emp2.employee_id AS emp_id_2,
    emp2.first_name AS first_name_2,
    emp2.last_name AS last_name_2
FROM employee_salary emp1
JOIN employee_salary emp2
  ON emp1.employee_id + 1 = emp2.employee_id;
```

## 🔹 Joining Multiple Tables

```sql id="j10"
-- Join demographics and salary
SELECT *
FROM employee_demographics dem
INNER JOIN employee_salary sal
  ON dem.employee_id = sal.employee_id;
```

---

```sql id="j11"
-- View department table
SELECT *
FROM parks_departments;
```

---

```sql id="j12"
-- Join three tables together
SELECT *
FROM employee_demographics dem
INNER JOIN employee_salary sal
  ON dem.employee_id = sal.employee_id
INNER JOIN parks_departments pdep
  ON sal.dept_id = pdep.department_id;
```

---

## 🧠 Key Notes

* `INNER JOIN` → only matching records
* `LEFT JOIN` → all rows from left + matched rows
* `RIGHT JOIN` → all rows from right + matched rows
* `SELF JOIN` → table joined with itself
* Use aliases (`dem`, `sal`) for better readability
* Always use `ON` condition to define relationship

---

## 🎯 Simple Understanding

* JOIN = combine data from multiple tables
* Based on a common column (usually `employee_id`)
