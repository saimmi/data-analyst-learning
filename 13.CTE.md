## 🔹 CTE (Common Table Expression)

---

## 📌 What is a CTE?

* A **temporary result set**
* Defined using `WITH`
* Used to make complex queries **more readable and reusable**
* Exists only during query execution

---

## 📌 Basic Syntax

```sql id="cte1"
WITH cte_name AS (
    SELECT column_name
    FROM table_name
    WHERE condition
)
SELECT *
FROM cte_name;
```

👉 CTE acts like a **temporary table**

---

## 🔹 1️⃣ Base JOIN Query

```sql id="cte2"
SELECT *
FROM employee_demographics dem
JOIN employee_salary sal
  ON dem.employee_id = sal.employee_id;
```

👉 Combines both tables for analysis

---

## 🔹 2️⃣ Aggregation Query

```sql id="cte3"
SELECT gender, 
       AVG(salary), 
       MAX(salary), 
       MIN(salary), 
       COUNT(salary)
FROM employee_demographics dem
JOIN employee_salary sal
  ON dem.employee_id = sal.employee_id
GROUP BY gender;
```

👉 Calculates metrics per gender

---

## 🔹 3️⃣ CTE with Aggregation

```sql id="cte4"
WITH CTE_example AS (
    SELECT gender, 
           AVG(salary) AS avg_sal, 
           MAX(salary) AS max_sal, 
           MIN(salary) AS min_sal, 
           COUNT(salary) AS count_sal
    FROM employee_demographics dem
    JOIN employee_salary sal
      ON dem.employee_id = sal.employee_id
    GROUP BY gender
)
SELECT AVG(avg_sal)
FROM CTE_example;
```

👉 First creates grouped data
👉 Then calculates **average of averages**

---

## 🔹 4️⃣ CTE with Column Aliases

```sql id="cte5"
WITH CTE_example (gender, avg_sal, max_sal, min_sal, count_sal) AS (
    SELECT gender, 
           AVG(salary), 
           MAX(salary), 
           MIN(salary), 
           COUNT(salary)
    FROM employee_demographics dem
    JOIN employee_salary sal
      ON dem.employee_id = sal.employee_id
    GROUP BY gender
)
SELECT *
FROM CTE_example;
```

👉 Column names defined directly in CTE declaration

---

## 🔹 5️⃣ Same Logic Using Subquery

```sql id="cte6"
SELECT AVG(avg_sal)
FROM (
    SELECT gender, 
           AVG(salary) AS avg_sal, 
           MAX(salary) AS max_sal, 
           MIN(salary) AS min_sal, 
           COUNT(salary) AS count_sal
    FROM employee_demographics dem
    JOIN employee_salary sal
      ON dem.employee_id = sal.employee_id
    GROUP BY gender
) AS example_subquery;
```

👉 Same result as CTE
👉 CTE is more readable

---

## 🔹 6️⃣ Filtering Data (Separate Queries)

```sql id="cte7"
SELECT employee_id, gender, birth_date 
FROM employee_demographics
WHERE birth_date > '1985-01-01';
```

```sql id="cte8"
SELECT employee_id, salary
FROM employee_salary
WHERE salary >= 50000;
```

---

## 🔹 7️⃣ Multiple CTEs + JOIN

```sql id="cte9"
WITH cte_example AS (
    SELECT employee_id, gender, birth_date 
    FROM employee_demographics
    WHERE birth_date > '1985-01-01'
),
cte_example2 AS (
    SELECT employee_id, salary
    FROM employee_salary
    WHERE salary >= 50000
)
SELECT *
FROM cte_example
JOIN cte_example2
  ON cte_example.employee_id = cte_example2.employee_id;
```

👉 Combines multiple CTEs
👉 Improves readability vs nested queries

---

## 🧠 Key Concepts

* `WITH` → defines CTE
* Can define **multiple CTEs**
* Improves readability
* Can replace subqueries
* Works well with:

  * JOINs
  * Aggregations
  * Filtering

---

## 🔁 CTE vs Subquery

| Feature     | CTE            | Subquery         |
| ----------- | -------------- | ---------------- |
| Readability | ✅ High         | ❌ Lower          |
| Reusability | ✅ Yes          | ❌ No             |
| Complexity  | ✅ Handles well | ❌ Hard to manage |

---

## ⚠️ Important Notes

* CTE exists only for **one query**
* Must be followed immediately by a `SELECT`
* Can be chained (multiple CTEs)
* Naming should be meaningful
