## 🔹 LIMIT & Aliasing

---

## 📌 View Full Table

```sql id="la1"
SELECT *
FROM employee_demographics;
```

👉 Displays all rows from the table

---

## 🔹 LIMIT (Restrict Number of Rows)

```sql id="la2"
-- Get first 3 rows
SELECT *
FROM employee_demographics
LIMIT 3;
```

---

```sql id="la3"
-- Get first 5 rows
SELECT *
FROM employee_demographics
LIMIT 5;
```

---

## 📌 LIMIT with ORDER BY

```sql id="la4"
-- Get top 3 oldest employees
SELECT *
FROM employee_demographics
ORDER BY age DESC
LIMIT 3;
```

👉 `ORDER BY` is applied before `LIMIT`

---

## 📌 LIMIT with Offset

```sql id="la5"
-- Skip first 3 rows, then return 1 row
SELECT *
FROM employee_demographics
ORDER BY age DESC
LIMIT 3, 1;
```

---

```sql id="la6"
-- Skip first 2 rows, then return 1 row
SELECT *
FROM employee_demographics
ORDER BY age DESC
LIMIT 2, 1;
```

👉 Syntax: `LIMIT offset, count`

---

## 🔹 Aliasing (Rename Columns)

```sql id="la7"
-- Using AS keyword
SELECT gender, AVG(age) AS avg_age
FROM employee_demographics
GROUP BY gender;
```

---

```sql id="la8"
-- Without AS (still works)
SELECT gender, AVG(age) avg_age
FROM employee_demographics
GROUP BY gender
HAVING avg_age > 40;
```

👉 Alias makes output more readable

---

## 🧠 Key Notes

* `LIMIT` controls number of rows returned
* `LIMIT offset, count` is used for pagination
* `ORDER BY` is applied before `LIMIT`
* `AS` is optional for aliasing
* Aliases help in:

  * readability
  * using names in `HAVING`

---

## 🎯 Simple Understanding

* `LIMIT` → control how many rows you see
* `ALIAS` → give a temporary name to columns
