## 🔹 Basic SELECT Queries

### 1️⃣ Select All Data

```sql
-- Using database name explicitly
SELECT *
FROM parks_and_recreation.employee_demographics;

-- Using current database (after USE statement)
SELECT *
FROM employee_demographics;
```

👉 Both queries return the same result
👉 Difference:

* First query uses **fully qualified table name**
* Second query works because database is already selected using `USE`

---

### 2️⃣ Select Specific Column

```sql
SELECT first_name
FROM employee_demographics;
```

👉 Returns only the `first_name` column
👉 Improves performance by selecting only required data

---

### 3️⃣ Select Multiple Columns + Calculated Column

```sql
SELECT first_name, 
       last_name, 
       birth_date, 
       age, 
       (age + 10) * 10 + 10 AS calculated_value
FROM employee_demographics;
```

👉 Adds a **calculated column** using arithmetic operations

### 🧠 Order of Execution (PEMDAS Rule)

* Parentheses `()`
* Exponents
* Multiplication `*`
* Division `/`
* Addition `+`
* Subtraction `-`

👉 Example:

* `(age + 10) * 10 + 10`
* First → `age + 10`
* Then → multiply by 10
* Then → add 10

---

### 4️⃣ DISTINCT Keyword

```sql
-- Unique first names
SELECT DISTINCT first_name
FROM parks_and_recreation.employee_demographics;

-- Unique genders
SELECT DISTINCT gender
FROM parks_and_recreation.employee_demographics;

-- Unique combinations of first_name and gender
SELECT DISTINCT first_name, gender
FROM parks_and_recreation.employee_demographics;
```

👉 `DISTINCT` removes duplicate values

### 🔍 Important Notes:

* `DISTINCT first_name` → removes duplicate names
* `DISTINCT gender` → returns only 'Male' and 'Female'
* `DISTINCT first_name, gender` → returns **unique combinations**, not individual uniqueness
